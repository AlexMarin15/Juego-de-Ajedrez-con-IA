Juego de Ajedrez con IA

Requerimientos del Sistema:

Objetivos del sistema:

Generar un juego de ajedrez con todos los posibles escenarios y sus reglas para poder tener un momento de disfrute personal jugando contra una IA que sea capaz de presentarnos una dificultad mediante el aprendizaje de patrones y los movimientos en el desarrollo del juego.

Requerimientos funcionales:

Requerimiento funcional 1: Concluir el juego ya sea por medio de tablas(ahogado) y jaque mate.
Requerimiento funcional 2: Poder mover las piezas a las casillas correspondientes de la pieza.
Requerimiento funcional 3: Capacidad de enrocar.
Requerimiento funcional 4: Capacidad de capturar al paso.
Requerimiento funcional 5: Poder realizar jaques y no poder mover hasta que el rey esté fuera de jaque.
Requerimiento funcional 6: Poder capturar piezas.
Requerimiento funcional 7: Que se muestre el tablero con las piezas completas.
Requerimiento funcional 8: No poder realizar movimientos que no son permitidos para la pieza.

Requerimientos no funcionales:

Requerimiento no funcional 1: Agilidad al realizar el movimiento(que no se quede mucho tiempo calculando el siguiente movimiento).
Requerimiento no funcional 2: El tablero debe de mostrar claramente los recuadros y cada pieza, así como los nombres y números de cada casilla.
Requerimiento no funcional 3: Que los movimientos que realice sean movimientos “lógicos”.
Requerimiento no funcional 4: Que pueda aprender de las jugadas de su rival.
Requerimiento no funcional 5: Que se pueda ajustar el nivel dependiendo del contrincante para que no gane de manera muy sencilla o muy difícilmente.

Herramienta para pruebas de Software
Pytest:

Se nos hizo mejor esta herramienta porque aparte de que es de código abierto se puede utilizar para pruebas de software de videojuegos que es nuestro caso, la manera en que se usa nos pareció sencilla porque al momento de hacer las pruebas nos sale un texto en las pruebas han sido aprobadas o pasadas.

También tuvimos en cuenta utilizar unittest y gitlab, pero son algo parecidos a pytest y decidimos utilizar pytest ya que es más nativo a python, es más fácil y factible. Además de que gitlab es un poco más laborioso.

Primero habíamos intentado usar la herramienta de Selenium pero al intentar hacer las pruebas de software nos dimos cuenta que solo servía para páginas web y entre otros, pero no para el videojuego de ajedrez.

Arbol de Fallos

Creately: Decidimos utilizar esta herramienta ya que tiene plantillas directas para la creación del diagrama de árbol de fallas, es una herramienta amigable, atractiva para trabajar y permite tener hasta 3 colaboradores a la hora de estar realizando un diagrama, además de ser una de las herramientas más utilizadas ya que cuenta con plantillas para lo que se necesite realizar, como en este caso un árbol de fallas. La utilización de la herramienta no es difícil, además de que cuenta con las opciones de poder exportar el diagrama al formato que necesitemos.
